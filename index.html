Flask Application - Natural Language Processing

MCQ Bot Using
RAG

The QuickMCQ Bot is a Flask-based application designed to generate multiple-choice questions (MCQs) on various topics such as Java, Python, and Machine Learning. It leverages the Retrieval-Augmented Generation (RAG) approach, using the Mistral 7B VO2 model from Hugging Face, to provide accurate and contextually relevant answers. The bot takes user inputs for the topic and the number of MCQs desired, then uses a FAISS vector store to access and process PDF files containing relevant information. It dynamically constructs MCQs with options and correct answers, presenting them in a user-friendly format. The bot's key components include Flask for the web framework, FAISS for vector storage, and Hugging Face for language model integration. It aims to enhance learning and assessment by providing a quick and easy way to generate MCQs for various topics.

Power BI Application - Data Visualization

Employee Attrition Analysis
Using Power BI

This Power BI project analyzes employee attrition to uncover trends and insights influencing employee turnover. The dataset was meticulously cleaned by removing null values, sorting in ascending order, eliminating duplicate employee IDs, and removing the 'Years at Current Manager' column. Key measures, such as attrition count and attrition rate, were calculated. Visualizations include cards displaying total employees, attrition count, attrition rate, average age, salary, and tenure; a donut chart for attrition by education field; clustered charts for attrition by age group, salary, job role, and gender; an area chart for attrition by years at the company; and a line chart for attrition rate by salary and gender. This analysis provides actionable insights to enhance employee retention strategies.

Power BI Application - Data Visualization

Bank Churn Analysis
Using Power BI

This Power BI project focuses on analyzing bank churn data to uncover factors influencing customer turnover. The dataset was sourced online and meticulously cleaned. Transformations included renaming columns, checking quality, and creating bins for age groups and credit scores. Duplicate tables were created for unique age and credit score bins. Data modeling was verified for accuracy. Key measures, such as total customers, lost customers, and churn rate, were calculated. Visualizations included cards for key metrics, pie charts for demographics, stacked column charts for product analysis, and various bar and column charts for churn analysis. Donut charts illustrated customers by country, churn status, and credit card holders. This project provides valuable insights to help improve customer retention and satisfaction.
